================================================================================
          GUIDE DE NAVIGATION : ARCHITECTURE DU PROJET SGP
================================================================================

Ce document t'aide à comprendre comment est organisé ton code source.
L'application suit le design pattern MVC (Modèle-Vue-Contrôleur).

--- 📂 RACINE DU PROJET --------------------------------------------------------
• pom.xml                : Fichier de configuration Maven (dépendances Spring).
• LANCER_APPLICATION.bat : Script pour démarrer l'appli facilement.
• bdd_initiale.sql       : Scripts SQL pour créer et remplir la base MySQL.
• presentation.txt       : Ton guide complet pour parler devant le jury.

--- 📂 BACKEND (Java) - src/main/java/com/kenfi/gestionpersonnel/ --------------

1. 🔵 MODELE (.modele)
   - Contient les "Objets" de l'application (Employe, Departement, Utilisateur).
   - C'est ici qu'on définit la structure des tables de la base de données.

2. 🟢 DEPOT (.depot) - [Repositories]
   - C'est la couche "Spring Data JPA".
   - Ces interfaces permettent de faire des SELECT, INSERT, UPDATE sans SQL.

3. 🟡 SERVICE (.service)
   - Contient la "Logique Métier".
   - C'est ici qu'on calcule la masse salariale ou qu'on valide les règles Gabonaises.

4. 🔴 CONTROLEUR (.controleur)
   - Reçoit les demandes de l'utilisateur (URL).
   - Fait le lien entre les Services et les vues Thymeleaf.

5. 🛡️ CONFIG (.config)
   - Contient la sécurité (ConfigSecurite.java).
   - C'est ici qu'on définit qui a accès à quelle page.

--- 📂 FRONTEND (Interface) - src/main/resources/ -----------------------------

1. 🖼️ TEMPLATES (./templates)
   - Contient les pages HTML (Thymeleaf).
   - layout.html : Le squelette commun à toutes les pages (Menu, barre de navigation).
   - tableau-de-bord.html : La page d'accueil avec les statistiques.

2. 🎨 STATIC (./static)
   - css/style.css : Tout le design visuel de l'application.

3. ⚙️ application.properties
   - La configuration du serveur et la connexion à la base MySQL.

================================================================================
💡 CONSEIL : Si tu veux modifier le design, va dans style.css. 
   Si tu veux ajouter une règle de calcul, va dans un Service.
